# AI Integration Setup for Medical Records App

## 🤖 AI-Powered Report Summarization

Your medical records app now includes AI-powered summarization of patient reports using OpenAI's GPT models.

### Features

- **Automatic Text Extraction**: Supports PDF, DOCX, TXT, and other document formats
- **Intelligent Summarization**: AI analyzes medical reports and creates comprehensive summaries
- **Doctor-Friendly Format**: Summaries highlight key findings, trends, and recommendations
- **Secure Processing**: All analysis happens server-side with proper error handling

### Setup Instructions

#### 1. Get OpenAI API Key

1. Go to [OpenAI Platform](https://platform.openai.com/)
2. Sign up or log in to your account
3. Navigate to API Keys section
4. Create a new API key
5. Copy the API key (keep it secure!)

#### 2. Environment Variables

**For Local Development:**
Create a `.env` file in the `backend/` directory:

```env
OPENAI_API_KEY=your_openai_api_key_here
NODE_ENV=development
```

**For Railway Deployment:**
1. Go to your Railway project dashboard
2. Navigate to Variables section
3. Add environment variable:
   - Name: `OPENAI_API_KEY`
   - Value: `your_openai_api_key_here`

#### 3. Install Dependencies

```bash
cd backend
npm install
```

#### 4. Test AI Integration

Create a `.env` file with your OpenAI API key, then run:

```bash
cd backend
node test-ai.js
```

You should see a sample medical summary generated by the AI.

### API Usage

The AI summarization is available via the existing endpoint:

```
POST /ai-summary/:patientEmail
```

**Request Body:**
```json
{
  "doctorEmail": "doctor@example.com"
}
```

**Response:**
```json
{
  "summary": "AI-generated comprehensive medical summary...",
  "documentCount": 3,
  "totalDocuments": 3
}
```

### Supported File Types

- **PDF**: Full text extraction using pdf-parse
- **DOCX**: Microsoft Word documents using mammoth
- **TXT**: Plain text files
- **Other**: Various formats using textract (images, RTF, etc.)

### Cost Considerations

- Uses GPT-3.5-turbo for cost efficiency

- Monitor your OpenAI usage dashboard for actual costs

### Security Notes

- API keys are stored securely as environment variables
- No patient data is sent to external services without processing
- All AI processing happens server-side
- Consider HIPAA compliance for medical data handling

### Troubleshooting

**"OpenAI API key not found"**
- Ensure `OPENAI_API_KEY` environment variable is set
- Check Railway environment variables if deployed

**"Could not extract text from file"**
- Verify file is not corrupted
- Check if file type is supported
- Some scanned PDFs may require OCR (consider tesseract.js for images)

**Rate limiting errors**
- OpenAI has rate limits; implement retry logic if needed
- Consider upgrading to paid tier for higher limits
